APPENDIX A

파이썬 설치 트러블슈팅

파이썬을 설치하다 보면 운영체제별로 자주 마주치는 함정이 있다. 이 부록은 Windows, macOS, Linux 환경에서 실제로 자주 보고되는 문제와 해결 방법을 모아 두었다. 1장의 설치 안내로 해결되지 않는 상황에서 사전처럼 찾아 보면 된다.

A.1 Windows 환경

PATH 환경 변수 누락 — python이라고 쳤더니 인식되지 않을 때

cmd나 PowerShell에 python을 입력했는데 'python' is not recognized as an internal or external command 메시지가 나오면 파이썬 실행 파일이 PATH에 등록되지 않은 상태이다. 가장 흔한 원인은 설치 마법사의 Add Python to PATH 체크 박스를 빠뜨린 것이다.

해결 방법은 두 가지다. 첫째, 파이썬 인스톨러를 다시 실행해 Modify를 누르고 Add python.exe to PATH 항목을 켜면 된다. 둘째, 수동으로 PATH를 추가한다. Win + R을 누르고 sysdm.cpl을 입력하면 시스템 속성이 열린다. 고급 → 환경 변수 → Path를 선택해 편집하고, 일반적으로 C:\Users\<사용자>\AppData\Local\Programs\Python\Python312 경로와 그 아래 Scripts 경로를 추가한다.

여러 파이썬 버전 충돌 — py launcher 사용

Anaconda·VS Code·또 다른 인스톨러로 여러 버전이 깔린 경우 python 명령이 어느 버전을 가리키는지 헷갈린다. Windows에는 py launcherPython Launcher for Windows가 함께 설치되므로 다음과 같이 명시적으로 호출한다.

py -3.12 --version       # 3.12 버전으로 실행
py -3.11 my_script.py    # 3.11 버전으로 스크립트 실행
py -0                    # 설치된 버전 목록

A.2 macOS 환경

시스템 파이썬과 사용자 파이썬 구분 — Homebrew 권장

macOS에는 시스템 파이썬(/usr/bin/python3)이 함께 설치되어 있지만, 시스템 도구가 의존하는 버전이라 직접 패키지를 설치하지 않는 편이 안전하다. 학습용으로는 Homebrew로 별도 설치하는 것이 일반적이다.

# Homebrew 설치(이미 깔려 있으면 생략)
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/\
Homebrew/install/HEAD/install.sh)"

# 파이썬 설치
brew install python@3.12

# 설치 위치 확인
which python3            # /opt/homebrew/bin/python3 (Apple Silicon)
                         # /usr/local/bin/python3     (Intel)

여러 버전 관리 — pyenv

학습이 진행되어 여러 프로젝트를 다루다 보면 프로젝트마다 다른 파이썬 버전이 필요해진다. 이 때 pyenv가 유용하다. 디렉터리별로 다른 버전을 자동 전환해 준다.

brew install pyenv
pyenv install 3.12.5         # 원하는 버전 설치
pyenv local 3.12.5           # 현재 폴더의 기본 버전 지정
python --version             # Python 3.12.5

SSL 인증서 오류 — Install Certificates.command

pip install 시 SSL: CERTIFICATE_VERIFY_FAILED 오류가 발생하면 Applications/Python 3.x/Install Certificates.command를 한 번 실행하면 해결된다.

A.3 Linux 환경

sudo pip install을 피하라

배포판 패키지 매니저로 깔린 시스템 파이썬에 sudo pip install 명령으로 패키지를 설치하면 시스템 도구가 의존하는 버전과 충돌해 OS가 망가질 수 있다. 대신 사용자 영역 또는 가상 환경을 쓴다.

# 사용자 영역에 설치 (--user)
pip install --user numpy

# 또는 venv로 가상 환경 만들기 (권장)
python3 -m venv .venv
source .venv/bin/activate     # 가상 환경 진입
pip install numpy             # sudo 없이 안전하게 설치
deactivate                     # 가상 환경 종료

python/python3 명령 혼동 (Ubuntu)

오래 된 Ubuntu에는 python이 python2를 가리키는 경우가 있다. python: command not found가 뜨거나 의도치 않은 버전이 실행되면 python3을 명시적으로 사용하자. 매번 입력이 번거롭다면 셸 별칭을 만든다.

# ~/.bashrc 또는 ~/.zshrc 끝에 추가
alias python=python3
alias pip=pip3

tkinter 모듈 누락 — IDLE이 실행되지 않을 때

Linux 배포판은 GUI 라이브러리(tkinter)를 별도 패키지로 분리해 두는 경우가 많다. IDLE 실행 시 ModuleNotFoundError: No module named 'tkinter'가 발생하면 다음을 설치한다.

# Debian/Ubuntu
sudo apt install python3-tk

# Fedora/RHEL
sudo dnf install python3-tkinter

A.4 공통 — pip 가속 — 미러 사용

해외 PyPI 서버 응답이 느리거나 차단된 환경에서는 국내 미러를 임시로 지정해 다운로드 속도를 끌어올릴 수 있다.

pip install -i https://mirror.kakao.com/pypi/simple/ numpy
pip install -i https://pypi.org/simple/ numpy   # 기본 PyPI

$HOME/.config/pip/pip.conf(또는 Windows의 %APPDATA%\pip\pip.ini)에 다음을 적어 두면 매번 옵션을 주지 않아도 된다.

[global]
index-url = https://mirror.kakao.com/pypi/simple/

도움이 되는 명령 한 줄 정리

  • python -V — 현재 파이썬 버전 확인
  • which python (Linux/macOS) 또는 where python (Windows) — 실제 실행 파일 경로
  • python -m site — 검색 경로(sys.path)와 사용자 디렉터리 확인
  • pip list — 설치된 패키지 목록
  • pip –version — pip가 어느 파이썬과 묶여 있는지 표시
