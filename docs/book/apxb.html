APPENDIX B

자주 보는 오류 메시지 사전

파이썬 인터프리터가 출력하는 오류 메시지는 처음에는 모두 같은 빨간색 글자로 보이지만, 역추적traceback의 마지막 줄에 적힌 예외 종류exception type와 메시지를 읽으면 원인을 거의 항상 짚어낼 수 있다. 이 부록은 입문자가 자주 마주치는 예외를 종류별로 모아, 각 오류마다 발생 원인과 해결 방법을 짧게 설명하였다. 사전처럼 찾아 보면 된다.

B.1 구문 오류 — 코드가 파이썬 문법에 맞지 않을 때

코드 자체가 파이썬 문법에 맞지 않아 프로그램 실행 전에 발견되는 오류이다. 코드를 수정해야 실행할 수 있다.

SyntaxError: invalid syntax

괄호·따옴표를 닫지 않았거나 들여쓰기 키워드(if, for, def) 뒤에 콜론(:)을 빠뜨린 경우에 발생한다. 메시지에 표시된 줄을 먼저 보고, 그 앞 줄에 미완성된 괄호가 없는지 확인한다. 줄 번호가 가리키는 자리에 진짜 문제가 있는 것이 아니라, 그 직전 줄의 닫히지 않은 괄호 때문에 다음 줄로 파싱이 흘러가다 멈춘 경우가 많다.

IndentationError: expected an indented block

함수·반복문의 본문이 비어 있거나 들여쓰기가 시작되지 않은 경우에 발생한다. 임시로 본문이 필요하지 않다면 pass 한 줄을 적어 비워 둔다. pass는 "아무 일도 하지 않는다"는 의미의 명령어이며, 자리만 차지하는 용도로 쓰인다.

IndentationError: unexpected indent / unindent does not match

공백과 탭 문자가 섞였거나 들여쓰기 폭이 줄마다 다른 경우에 발생한다. 사람 눈에는 같은 들여쓰기처럼 보여도 파이썬은 두 종류를 다르게 인식한다. 에디터의 Show Whitespace(또는 Render Whitespace) 옵션을 켜면 실제 문자가 보이며, 모든 들여쓰기를 4칸 공백으로 통일하면 사라진다.

B.2 이름·범위 오류

NameError: name 'x' is not defined

정의되지 않은 변수·함수를 사용하면 발생한다. 가장 먼저 의심해야 할 것은 철자와 대소문자이다. 파이썬은 Print와 print, List와 list를 다른 이름으로 본다. 다른 모듈에 있는 함수를 쓰려고 했다면 파일 맨 위에 import 문을 빼먹지 않았는지도 확인한다.

UnboundLocalError: local variable 'x' referenced before assignment

함수 안에서 같은 이름의 변수에 대입한 줄이 있어 파이썬이 그 변수를 지역 변수로 간주했지만, 그 줄보다 앞에서 값을 읽었을 때 발생한다. 함수 어디서든 x = ...이라고 적으면 파이썬은 x를 그 함수의 지역 변수로 처음부터 끝까지 취급한다. 전역 변수를 함수 안에서 변경하려면 함수 첫 줄에 global x를 적어야 한다.

B.3 타입·값 오류

TypeError: unsupported operand type(s) for +: 'int' and 'str'

서로 다른 자료형을 직접 연산할 때 발생한다. age = input('나이?')처럼 입력받은 값은 str이므로, 숫자처럼 더하려면 int(input(\dots{}))로 명시적 형 변환을 해 주거나 f-string(f'{age}살입니다')으로 문자열에 끼워 넣는다.

TypeError: 'tuple' object does not support item assignment

튜플은 불변 객체immutable이므로 t[0] = 5처럼 항목을 바꿀 수 없다. 항목을 자주 바꿔야 하는 데이터라면 처음부터 리스트([\dots{}])를 사용한다. 한두 번 부분만 바꿔야 한다면 새 튜플을 만들어 다시 대입하는 방식을 쓴다(t = (5,) + t[1:]).

ValueError: invalid literal for int() with base 10: 'abc'

숫자가 아닌 문자열을 int()로 변환하려 할 때 발생한다. 사용자 입력에 자주 등장하는 오류이므로, 입력값을 먼저 isdigit()으로 검증하거나 try-except ValueError로 잡아 다시 입력받는 패턴이 안전하다.

B.4 인덱스·키 오류

IndexError: list index out of range

리스트 길이를 벗어나는 인덱스에 접근할 때 발생한다. 길이를 알고 싶다면 len(lst)와 비교하고, 음수 인덱스(-1, -2)로 마지막부터 접근할 때도 길이를 넘어선 음수면 같은 오류가 발생한다. 반복문에서는 range(len(lst)) 대신 enumerate(lst)를 쓰면 인덱스를 직접 다루는 실수를 줄일 수 있다.

KeyError: 'name'

딕셔너리에 존재하지 않는 키를 조회할 때 발생한다. 키 존재 여부가 불확실하다면 d['name'] 대신 d.get('name')을 쓰자. 키가 없을 때 None을 돌려주며, 두 번째 인자로 기본값을 줄 수도 있다(d.get('name', 'unknown')).

B.5 속성 오류

AttributeError: 'str' object has no attribute 'append'

호출하려는 메소드를 그 자료형이 지원하지 않는 경우에 발생한다. 변수 안에 들어 있는 객체의 종류를 type(x)로 먼저 확인하자. append()는 리스트의 메소드이므로 문자열에는 없다. 문자열에 새 글자를 이어 붙일 때는 + 연산자나 join()을 사용한다.

AttributeError: module 'xxx' has no attribute 'yyy'

모듈에 그런 이름의 함수가 없거나, 자기 작업 폴더에 표준 모듈과 같은 이름의 파일(random.py, math.py)을 만들어 표준 모듈을 가리는 경우에 자주 발생한다. 파일 이름을 my_random.py처럼 바꾸고, 같은 폴더의 __pycache__ 폴더와 *.pyc 파일을 정리한 뒤 다시 실행한다.

B.6 파일·입출력 오류

FileNotFoundError: [Errno 2] No such file or directory: 'data.txt'

파일이 현재 작업 디렉터리에 없을 때 발생한다. 작업 디렉터리는 코드 파일이 있는 폴더와 다를 수 있으므로, import os; print(os.getcwd())로 먼저 현재 위치를 확인한다. 가장 안전한 방법은 절대 경로를 쓰거나, Path(__file__).parent / 'data.txt'처럼 코드 파일 기준 상대 경로를 사용하는 것이다.

PermissionError: [Errno 13] Permission denied

읽기 전용 파일에 쓰기 모드('w', 'a')로 열거나, 보호된 디렉터리(예: C:\Program Files)에 새 파일을 쓰려고 할 때 발생한다. 작업 폴더를 사용자의 Documents나 홈 디렉터리로 옮기면 대부분 해결된다. 다른 프로그램이 같은 파일을 열고 있을 때도 같은 오류가 날 수 있으니 그 프로그램을 닫아 보자.

UnicodeDecodeError: 'utf-8' codec can't decode byte

파일의 인코딩과 디코더가 일치하지 않을 때 발생한다. 한글이 포함된 윈도 파일은 흔히 cp949로 저장되어 있다. open(path, encoding='cp949')로 시도하거나, BOM이 붙은 UTF-8이라면 encoding='utf-8-sig'로 열어 보자.

B.7 모듈·임포트 오류

ModuleNotFoundError: No module named 'numpy'

패키지가 설치되어 있지 않거나, 설치된 파이썬과 실행 중인 파이썬이 서로 다른 경우에 발생한다. 가장 안전한 설치 명령은 python -m pip install numpy로, "지금 실행 중인 그 파이썬"으로 설치하는 것이다. 가상 환경(venv, conda) 안에서 작업한다면 그 환경이 활성화되어 있는지 먼저 확인한다.

ImportError: cannot import name 'X' from 'Y'

모듈 Y는 있지만 그 안에 X라는 이름이 없는 경우에 발생한다. 라이브러리가 새 버전으로 올라가면서 함수가 다른 위치로 옮겨지거나 사라진 경우가 많으니, pip show <lib>로 버전을 확인하고 공식 문서에서 새 임포트 경로를 찾아 본다.

B.8 산술·재귀·메모리

ZeroDivisionError: division by zero

/ 또는 % 연산의 분모가 (0)일 때 발생한다. 분모를 미리 if denom == 0:로 검사해 안전하게 처리하거나 try-except ZeroDivisionError로 잡는다.

RecursionError: maximum recursion depth exceeded

재귀 깊이가 약 1 000회를 넘어 호출 스택이 한계에 도달했을 때 발생한다. 기저 조건(base case)이 도달 가능한지 다시 점검하자. 근본적으로는 같은 작업을 반복문(for/while)으로 바꾸거나, 같은 부분 문제를 여러 번 푸는 경우라면 메모이제이션으로 캐싱한다(4장 재귀호출 절 참고).

MemoryError

리스트·문자열이 가용 메모리를 초과할 때 발생한다. 큰 파일을 f.read()로 한 번에 읽기보다는 for line in file:로 한 줄씩 처리하고, 큰 시퀀스는 generator로 만들어 한 번에 메모리에 올리는 양을 줄인다.

역추적 읽기 요령

  • 역추적의 가장 아래 줄에 있는 ExceptionType: message가 핵심이다. 위로 갈수록 그 호출이 일어난 단계이다.
  • 자기가 작성한 코드의 줄 번호를 먼저 확인하고, 그 줄과 바로 앞 줄을 모두 살핀다.
  • 메시지에 변수 이름·숫자가 그대로 적히는 경우가 많다. 따옴표 안의 문자열을 단서로 쓴다.
